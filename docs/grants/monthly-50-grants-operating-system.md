## 1. System Overview

This operating system turns grant hunting from ad-hoc submissions into a **repeatable monthly machine**. The goal is not volume for its own sake — it is **statistical coverage** of the non-dilutive funding landscape while preserving founder bandwidth through tiered effort allocation.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    MONTHLY 50 GRANTS ENGINE                          │
├─────────────────────────────────────────────────────────────────────┤
│  DISCOVER (Week 1) → SCORE (ongoing) → APPLY (Weeks 2–4)         │
│       ↓                    ↓                    ↓                    │
│  Source 60+ opps      grant-scoring-model    5 + 15 + 30 split     │
│  Score ≥6 only        marketing-capital-     weekly workflow        │
│                       narrative.md           daily agent prompts     │
│       ↓                    ↓                    ↓                    │
│  FOLLOW-UP (Week 4 + rolling) → REPORT KPIs → REFINE next month   │
└─────────────────────────────────────────────────────────────────────┘
```

### Core principle

Frame all growth spend as **liquidity activation** — campaigns that bring verified sellers and capital-ready buyers into a live marketplace — never as generic "advertising budget." See `marketing-capital-narrative.md`.

## 8. Quality Gates — Do Not Submit If

| Gate | Rule |
|------|------|
| Score floor | Composite score < 6.0 (see `grant-scoring-model.md`) |
| Eligibility | US LLC explicitly ineligible — unless partner path documented |
| Marketing | If cash grant and marketing **not allowed** — reallocate narrative to "market research" only |
| Dilution | Accelerator requires >15% equity default — downgrade to Tier C or skip |
| Duplicate | Same program applied within reapply window |
| Evidence | Tier A without live URL demo ready — hold |
